Manual Setup Guide: SharePoint Ingestion SPNs/App Registrations

Purpose: handover guide for administrators to manually set up Entra app registrations, service principals, permissions, and Key Vault secrets for this project in another environment.

Current runtime requirement: the Python application authenticates through Microsoft Graph and requires Microsoft Graph → Application → Sites.ReadWrite.All with admin consent.

1) Architecture summary

  • Create one app registration + service principal per environment (for example, dev and prod).
  • Recommended display names:
    • spn-sharepoint-ingest-dev
    • spn-sharepoint-ingest-prod
  • SharePoint Online and Microsoft Graph are separate resource APIs in Entra ID.
  • The same app can be granted permissions to both APIs if needed.

3) Create app registration and service principal (per environment)

Option A: Azure Portal

  1. Go to Microsoft Entra admin center → App registrations → New registration.
  2. Name it (for example spn-sharepoint-ingest-dev).
  3. Supported account type: Accounts in this organizational directory only.
  4. Create the app.
  5. Open Certificates & secrets → New client secret and copy the secret value immediately.
  6. Open Overview and record:
    • Application (client) ID
    • Directory (tenant) ID
  7. Ensure an Enterprise Application (service principal) exists automatically for this app.

Option B: Azure CLI

az login
az ad app create --display-name "spn-sharepoint-ingest-dev" --sign-in-audience AzureADMyOrg

# Save appId from output, then create the service principal:
az ad sp create --id <APP_ID>

# Create a client secret (capture returned password safely):
az ad app credential reset --id <APP_ID> --append --years 2 --query password -o tsv

4) Configure required API permission (current runtime)

For each app registration, add this permission:

  • API: Microsoft Graph (00000003-0000-0000-c000-000000000000)
  • Type: Application permission
  • Permission: Sites.ReadWrite.All

Then click Grant admin consent for the tenant. 

# Optional CLI verification (Graph app role assignment)
SP_ID=$(az ad sp list --filter "appId eq '<APP_ID>'" --query "[0].id" -o tsv)
az rest --method GET \
  --url "https://graph.microsoft.com/v1.0/servicePrincipals/$SP_ID/appRoleAssignments" \
  --query "value[?appRoleId=='9492366f-7969-46a4-8d15-ed1a20078fff']"

5) Optional SharePoint Online permission model (only if your target setup requires it)

If your organization wants explicit SharePoint app permissions/site grants, you may also configure:

  • API: Office 365 SharePoint Online (00000003-0000-0ff1-ce00-000000000000)
  • Type: Application permission
  • Permission: Sites.Selected

Grant admin consent, then (optionally) grant site-level write access with PnP PowerShell:

Connect-PnPOnline -Url "https://<tenant>-admin.sharepoint.com" -Interactive
Grant-PnPAzureADAppSitePermission -AppId "<APP_ID>" -DisplayName "spn-sharepoint-ingest-dev" -Site "<SITE_URL>" -Permissions Write
Get-PnPAzureADAppSitePermission -Site "<SITE_URL>"
This step is optional for the current Python Graph-based runtime, but can be kept for governance or future compatibility.

6) Create Key Vault secrets (per environment)

Create the following secrets in your target Key Vault for each environment:

Secret name patternExample value
dm-sharepoint-<env>-client-idApp registration client ID
dm-sharepoint-<env>-client-secretClient secret value
dm-sharepoint-<env>-tenant-idTenant ID
dm-sharepoint-<env>-site-urlSharePoint site URL for that env
dm-sql-<env>-serverSQL server/instance
dm-sql-<env>-databaseSQL database name
 
# Example CLI secret creation
az keyvault secret set --vault-name <VAULT_NAME> --name dm-sharepoint-dev-client-id --value <APP_ID>
az keyvault secret set --vault-name <VAULT_NAME> --name dm-sharepoint-dev-client-secret --value <APP_SECRET>
az keyvault secret set --vault-name <VAULT_NAME> --name dm-sharepoint-dev-tenant-id --value <TENANT_ID>
az keyvault secret set --vault-name <VAULT_NAME> --name dm-sharepoint-dev-site-url --value "https://contoso.sharepoint.com/sites/data_ingest_dev"
az keyvault secret set --vault-name <VAULT_NAME> --name dm-sql-dev-server --value "localhost:1433"
az keyvault secret set --vault-name <VAULT_NAME> --name dm-sql-dev-database --value "ingest_dev"

7) Validation checklist

  • ☐ App registration created for each environment.
  • ☐ Service principal exists for each app.
  • ☐ Client secret created and captured securely.
  • ☐ Microsoft Graph Sites.ReadWrite.All added and admin consent granted.
  • ☐ Required Key Vault secrets created for each environment.
  • ☐ SharePoint connectivity test succeeds.
